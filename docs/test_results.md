# ScanMate 測試結果

本文件記錄目前期末展示版的實測與待補量測項目。ScanMate 屬於靜態文件掃描 App，不是連續影像串流辨識系統，因此 FPS 不作為主要成功指標。

## 已完成實測

| 測試項目 | 方法 | 結果 |
| --- | --- | --- |
| Gradle build | 執行 `.\gradlew.bat assembleDebug` | 通過 |
| APK 安裝 | 使用 ADB 安裝 debug APK | 通過 |
| App 啟動 | ADB 啟動 MainActivity | 通過，無 FATAL EXCEPTION |
| 首頁入口 | UIAutomator 檢查 8 個首頁功能 ID | 通過 |
| AI 學習助理入口 | 從首頁點擊 AI 學習助理 | 通過，無 OCR 文字時顯示空狀態 |

## 展示前建議再測

| 流程 | 測試資料 | 成功條件 |
| --- | --- | --- |
| 單頁掃描 | 1 張講義或筆記照片 | 可裁切、預覽、儲存 1 頁 PDF |
| 多頁掃描 | 3 張文件照片 | 可儲存 3 頁 PDF，最近文件顯示頁數 |
| PDF 分享 | 任一已儲存 PDF | 系統分享選單可正常出現 |
| PDF 開啟 | 任一已儲存 PDF | 使用系統 PDF viewer 開啟 |
| OCR | 清楚印刷文字圖片 | 可顯示 OCR 結果 |
| OCR 到 AI | OCR 完成後點擊送到 AI 學習助理 | 顯示原文、類型、命名建議、重點、關鍵字、複習題 |
| 失敗案例 | 模糊、反光或背景複雜文件 | 可說明限制，App 不應閃退 |

## 效能與正確率說明

| 指標 | 目前狀態 |
| --- | --- |
| FPS | 不適用，系統不做連續影像即時辨識 |
| Latency | 受圖片解析度、手機效能與 OpenCV 處理影響；展示時建議使用清楚、解析度適中的圖片 |
| OCR Accuracy | 未建立正式標註資料集，展示版以功能驗證為主 |
| 文件偵測成功率 | 未建立正式標註資料集，展示版以多種拍攝條件人工驗證 |

後續若要讓報告更完整，建議準備 20 到 50 張測試圖片，分成清楚、模糊、斜拍、暗光、背景複雜等類別，記錄文件偵測成功率、OCR 可讀率與平均處理時間。
